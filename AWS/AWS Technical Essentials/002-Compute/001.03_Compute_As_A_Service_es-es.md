# AWS Compute como Servicio

## Descripción General
AWS ofrece tres tipos fundamentales de opciones de cómputo:
- **Máquinas Virtuales (VMs)**
- **Servicios de Contenedores**
- **Cómputo Serverless**

## La Necesidad de Cómputo
Cada empresa requiere capacidad de cómputo para ejecutar aplicaciones, que pueden incluir:
- Servidores web
- Tareas por lotes
- Bases de datos
- Software de recursos humanos
- Aprendizaje automático

Tradicionalmente, el tiempo, dinero y esfuerzo necesarios para configurar recursos de cómputo locales son significativos.

## Servidores: El Bloque Fundamental de las Aplicaciones
El primer bloque necesario para alojar una aplicación es un **servidor**:
- Los servidores manejan solicitudes HTTP y envían respuestas siguiendo el modelo cliente-servidor.
- La comunicación basada en API también se incluye en este modelo.

### Modelo Cliente-Servidor
- **Cliente**: Una persona o computadora que envía una solicitud.
- **Servidor**: Una computadora, o conjunto de computadoras, conectada a internet que provee sitios y aplicaciones a los usuarios.

### Capacidades de los Servidores
Los servidores proveen:
- **CPU**: Potencia de cómputo para procesar.
- **Memoria**: Almacenamiento de datos para aplicaciones y solicitudes.
- **Red**: Capacidad para manejar la comunicación con los usuarios.

### Ejemplos de Servidores HTTP
- **Opciones para Windows**: Internet Information Services (IIS)
- **Opciones para Linux**: Apache HTTP Server, Nginx, Apache Tomcat

Para ejecutar un servidor HTTP en AWS, debes elegir un servicio de cómputo en la Consola de Administración de AWS, donde se enumeran todos los servicios de cómputo disponibles.

## Desafíos de la Infraestructura Local
Poseer servidores físicos implica:
1. **Investigación y Compra**: Determinar qué tipo y cuántos servidores comprar.
2. **Tiempos de Entrega**: Los servidores suelen tardar semanas o meses en llegar.
3. **Instalación**: Los servidores deben instalarse, apilarse y cablearse en un centro de datos.
4. **Configuración y Mantenimiento**: Los servidores requieren protección, alimentación y mantenimiento continuo.
5. **Inversión Rígida**: Una vez comprados, estás atado a los servidores, se utilicen o no.

Escalar la infraestructura física rápidamente es un desafío, y recuperar inversiones no utilizadas es prácticamente imposible.

## Ventajas del Cómputo como Servicio
AWS simplifica las operaciones al encargarse de:
- Configuración y seguridad de centros de datos.
- Adquisición, instalación y mantenimiento de hardware.

AWS ofrece capacidad de cómputo disponible en línea, lo que permite a las empresas enfocarse en sus aplicaciones en lugar de la infraestructura.

### Beneficios del AWS Compute
1. **El Cómputo Adecuado para Tus Cargas de Trabajo**: Soluciones personalizadas para diversas cargas de trabajo, asegurando un rendimiento óptimo y eficiencia en costos.
2. **Seguridad Incorporada**: Funciones de seguridad robustas, incluido AWS Nitro System, y cumplimiento con estándares como PCI-DSS, HIPAA y GDPR.
3. **Flexibilidad para Optimizar Costos**: Modelos de precios como EC2 Spot Instances y Savings Plans ofrecen ahorros significativos.
4. **Cómputo Donde lo Necesitas**: AWS se extiende a ubicaciones periféricas y entornos locales con servicios como AWS Outposts y AWS Local Zones.

## Servicios Principales de Cómputo en AWS

### Amazon EC2: Máquinas Virtuales
Amazon Elastic Compute Cloud (Amazon EC2) es un servicio web que proporciona capacidad de cómputo segura y redimensionable en la nube. Con EC2, puedes aprovisionar servidores virtuales llamados **EC2 instances**. En segundo plano:
- AWS gestiona las máquinas host y la capa de hypervisor.
- AWS instala el sistema operativo de la máquina virtual, conocido como **guest operating system**.

### Otras Opciones de Cómputo
Además de máquinas virtuales como EC2, AWS ofrece:
1. **Servicios de Contenedores**: Ideales para administrar aplicaciones containerizadas.
2. **Cómputo Serverless**: Perfecto para construir aplicaciones optimizadas para escalabilidad y eficiencia.

---

### El AWS Nitro System
AWS reinventó la infraestructura de virtualización con AWS Nitro System, que incluye:
- **Nitro Cards**: Desplazan y aceleran funciones de I/O para mejorar el rendimiento.
- **Nitro Security Chip**: Mejora la seguridad delegando funciones críticas a hardware dedicado y prohibiendo el acceso administrativo.
- **Nitro Hypervisor**: Un hypervisor liviano que gestiona la asignación de memoria y CPU, proporcionando un rendimiento casi bare-metal.

---

### Cómo Elegir la Opción de Cómputo Correcta
Al configurar servidores en AWS, debes elegir el servicio de cómputo adecuado para tu caso de uso. Existen tres tipos fundamentales de opciones de cómputo:
1. **Máquinas Virtuales (VMs)**:
   - Emula un servidor físico y permite la instalación de un servidor HTTP.
   - Requiere un **hypervisor** (software o firmware) para gestionar recursos de hardware físico entre máquinas virtuales.
2. **Servicios de Contenedores**:
   - Adecuados para aplicaciones containerizadas.
3. **Cómputo Serverless**:
   - Ideal para aplicaciones escalables y basadas en eventos.

Algunos servicios de cómputo de AWS, como servicios de contenedores y opciones serverless, están construidos sobre EC2 o conceptos de virtualización. Entender Amazon EC2 es un paso fundamental antes de avanzar hacia estas opciones más avanzadas.
