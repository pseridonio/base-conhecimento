# AWS Compute como Serviço

## Visão Geral
A AWS oferece três tipos fundamentais de opções de computação:
- **Máquinas Virtuais (VMs)**
- **Serviços de Containers**
- **Computação Serverless**

## A Necessidade de Computação
Toda empresa precisa de capacidade de computação para executar aplicativos, que podem incluir:
- Servidores web
- Tarefas em lote
- Bancos de dados
- Software de RH
- Aprendizado de máquina

Tradicionalmente, o tempo, dinheiro e esforço necessários para configurar recursos de computação no local são significativos.

## Servidores: O Bloco de Construção de Aplicativos
O primeiro componente necessário para hospedar um aplicativo é um **servidor**:
- Servidores lidam com solicitações HTTP e enviam respostas seguindo o modelo cliente-servidor.
- A comunicação baseada em APIs também segue esse modelo.

### Modelo Cliente-Servidor
- **Cliente**: Uma pessoa ou computador que envia uma solicitação.
- **Servidor**: Um computador, ou conjunto de computadores, conectado à internet que fornece sites e aplicativos aos usuários.

### Capacidades de Servidores
Os servidores fornecem:
- **CPU**: Poder de computação para processamento.
- **Memória**: Armazenamento de dados para aplicativos e solicitações.
- **Rede**: Capacidade para lidar com a comunicação com os usuários.

### Exemplos de Servidores HTTP
- **Opções para Windows**: Internet Information Services (IIS)
- **Opções para Linux**: Apache HTTP Server, Nginx, Apache Tomcat

Para executar um servidor HTTP na AWS, você precisa escolher um serviço de computação no Console de Gerenciamento da AWS, onde todos os serviços de computação disponíveis são listados.

## Desafios da Infraestrutura Local
Possuir servidores físicos envolve:
1. **Pesquisa e Compra**: Determinar o tipo e a quantidade de servidores para comprar.
2. **Tempo de Espera de Entrega**: Os servidores geralmente levam semanas ou meses para chegar.
3. **Instalação**: Os servidores devem ser instalados, empilhados e conectados em um data center.
4. **Configuração e Manutenção**: Os servidores precisam ser protegidos, energizados e mantidos regularmente.
5. **Investimento Rígido**: Após a compra, você está preso aos servidores, sejam eles utilizados ou não.

Escalar infraestrutura física rapidamente é desafiador e recuperar investimentos não utilizados é praticamente impossível.

## Vantagens da Computação como Serviço
A AWS simplifica as operações ao lidar com:
- Configuração e segurança de data centers.
- Aquisição, instalação e manutenção de hardware.

A AWS oferece capacidade de computação prontamente disponível online, permitindo que as empresas se concentrem em seus aplicativos em vez da infraestrutura.

### Benefícios do AWS Compute
1. **Computação Correta para Suas Cargas de Trabalho**: Soluções personalizadas para diversas cargas de trabalho garantem desempenho ideal e eficiência de custos.
2. **Segurança Integrada**: Recursos de segurança robustos incluem o AWS Nitro System e conformidade com padrões como PCI-DSS, HIPAA e GDPR.
3. **Flexibilidade para Otimizar Custos**: Modelos de precificação como EC2 Spot Instances e Savings Plans oferecem economia significativa.
4. **Computação Onde Você Precisa**: A AWS se estende a locais de borda e ambientes locais com serviços como AWS Outposts e AWS Local Zones.

## Serviços de Computação Principais na AWS

### Amazon EC2: Máquinas Virtuais
Amazon Elastic Compute Cloud (Amazon EC2) é um serviço web que fornece capacidade de computação segura e redimensionável na nuvem. Com o EC2, você pode provisionar servidores virtuais chamados **EC2 instances**. Nos bastidores:
- A AWS gerencia as máquinas host e a camada do hypervisor.
- A AWS instala o sistema operacional da máquina virtual, chamado de **guest operating system**.

### Outras Opções de Computação
Além de máquinas virtuais como EC2, a AWS fornece:
1. **Serviços de Containers**: Ideais para gerenciar aplicativos containerizados.
2. **Computação Serverless**: Perfeita para construir aplicativos otimizados para escalabilidade e eficiência.

---

### O AWS Nitro System
A AWS reinventou a infraestrutura de virtualização com o AWS Nitro System, que inclui:
- **Nitro Cards**: Deslocam e aceleram funções de I/O para melhorar o desempenho.
- **Nitro Security Chip**: Melhora a segurança ao delegar funções críticas a hardware dedicado e proibir acesso administrativo.
- **Nitro Hypervisor**: Um hypervisor leve que gerencia a alocação de memória e CPU, proporcionando desempenho próximo ao bare-metal.

---

### Escolhendo a Opção de Computação Correta
Ao configurar servidores na AWS, você deve escolher o serviço de computação correto para seu caso de uso. Existem três tipos fundamentais de opções de computação:
1. **Máquinas Virtuais (VMs)**:
   - Emula um servidor físico e permite a instalação de um servidor HTTP.
   - Requer um **hypervisor** (software ou firmware) para gerenciar recursos de hardware físico em várias máquinas virtuais.
2. **Serviços de Containers**:
   - Adequados para aplicativos containerizados.
3. **Computação Serverless**:
   - Ideal para aplicativos escaláveis e orientados a eventos.

Alguns serviços de computação da AWS, como serviços de containers e opções serverless, são construídos sobre EC2 ou conceitos de virtualização. Entender o Amazon EC2 é um passo fundamental antes de avançar para essas opções mais avançadas.
